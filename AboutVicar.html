Welcome to Abbot!

Abbot is a tool designed to convert dissimilar collections of XML texts into a common interoperable form. Abbot's key feature is the ability to read an XML schema file and output procedures to convert source files into a valid form that is consistent with the target schema. Abbot's schema-harvesting procedures focus on TEI, but are extremely flexible and format agnostic. Abbot makes no particular judgment or demand concerning the type of interoperability sought. It can transform texts into a variety of TEI schema, and accommodates user customization.

Abbot is more likely than conventional file-conversion methods to spot and deal with problems because it operates consistently and algorithmically across large numbers of texts. Abbot sets its course on an ambitious but sensible path—moving toward total interoperability, while at the same time accepting the uniqueness of individual text collections. Abbot's method allows for different forms of interoperability, from small, one-off instances to the creation of large, permanent digital libraries.

Abbot was developed at the Center for Digital Research in the Humanities by Brian L. Pytlik Zillig, Stephen Ramsay, Martin Mueller, and Frank Smutniak. Support for Abbot was provided by the Andrew W. Mellon Foundation.

Access

In order for your data files to persist between sessions, you are required to sign in.

You may sign in with a Vicar account or use an account from either yahoo.com or gmail.com.

Create a local account directly with Vicar by clicking on the REGISTER link and entering your email address and a new Vicar-specific password. You will receive a confirmation link via email. Click on this link to activate your account.

If you have a yahoo.com or gmail.com account, you can bypass Vicar registration and use one of these accounts instead. Clicking on the respective links allows you to sign into your yahoo.com or gmail.com account and have the respective site relay verification to Vicar.

Access to your data collections is tied to your email address. You must sign in using the same method each time.

Operation

You can create collections and upload your input and schema files once you are signed in. Click on CREATE NEW COLLECTION and enter a name containing only letters a-z and A-Z with no spaces. Press SAVE to create your collection. Since it is new, it will be empty. Clicking on COLLECTIONS will show a list of your collections. Select a collection name to see its contents and add files.

You can add individual files or multiple files at once if such actions are supported by your browser (see Notes). You can also upload a zip file containing multiple XML data files. Click on the UNZIP link and the contents of the zip file will be unpacked into the collection.

Once you have files in your collection, you will see a dropdown menu with schema names and a button with three green 'greater than' arrows. The schema list will show Vicar-provided schema names listed in blue. User-supplied schema names appear at the end of the list in black.

Select a schema and then click on the arrow button to invoke file conversion. Vicar will show the progress of the Abbot conversion and will populate an output directory with the result of its run. Each file will, to its right, have a green OK in a green outline or a red number in a red outline. This is the result of schema validation. The red number is the number of errors found by the validation software. Clicking on the red number will show the errors reported from the validation stage. An OK indicates that the file is valid.

Click on an individual XML output file to display its contents and save it to your local computer. If the output contains multiple files you will be given an option to package and download them as a single zip file.

Your files will persist in your collections until you delete them or until your account is inactive for a duration to be determined. Delete your collection and its contents by clicking on the red X in COLLECTIONS.

Notes

Vicar works best with most up-to-date browsers which support HTML5. HTML5 compatibility is needed for multiple file uploads. Single uploads will still be possible without it.

Vicar works best with javascript enabled. A browser which has javascript disabled will not allow mutliple uploads nor will it display progress bars related to uploads or conversions but should otherwise be functional.

During this beta stage no single uploaded input file can exceed 15MB.
